# fix-f423d23536860de2 — RedisSessionStorage.readAllKeys() returns AsyncIterableIterator — likely incompatible with grammY StorageAdapter

**Weight:** 0.0000 (share of project budget)
**Reward:** 0 RDX

`src/toolkit/session/redis.ts:63` declares `async *readAllKeys(): AsyncIterableIterator<string>`. grammY's `StorageAdapter` interface expects `readAllKeys(): MaybePromise<IterableIterator<string>>`. An `AsyncGenerator` (returned by `async function*`) is NOT a synchronous `IterableIterator` — it has `Symbol.asyncIterator` but not `Symbol.iterator`. If grammY's session middleware does `const keys = await storage.readAllKeys(); for (const k of keys)` (synchronous iteration), this will throw `TypeError: keys is not iterable` at runtime in production when `REDIS_URL` is set. The isolated unit test at `test/session-redis.test.ts:50` works only because it uses `for await...of`. No integration test validates Redis adapter compatibility with grammY's session middleware.
